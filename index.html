SuperClaude란 무엇인가?

SuperClaude는 Claude Code를 위한 고급 프레임워크로, AI 기반 개발을 위한 포괄적인 도구 세트를 제공합니다.

설치 및 설정

SuperClaude를 시작하기 위한 단계별 설정 가이드입니다.

# 1. Claude Code 설치 (이미 설치되어 있다면 생략)
npm install -g @anthropic/claude-code

# 2. SuperClaude 프레임워크 파일을 ~/.claude/ 디렉토리에 복사
mkdir -p ~/.claude
cp CLAUDE.md COMMANDS.md FLAGS.md PRINCIPLES.md RULES.md ~/.claude/

# 3. 환경 설정 확인
claude --version
💡 팁: CLAUDE.md 파일은 모든 SuperClaude 설정의 진입점입니다. 이 파일을 통해 전체 프레임워크가 활성화됩니다.

명령어 시스템

SuperClaude의 50개 이상 명령어는 개발 워크플로우의 모든 단계를 커버합니다.

🔨 개발 명령어

/build 프로젝트 빌드 및 프레임워크 감지
/implement 기능 및 코드 구현
/design 시스템 및 컴포넌트 설계

🔍 분석 명령어

/analyze 다차원 코드 및 시스템 분석
/troubleshoot 문제 조사 및 해결
/explain 교육적 설명 제공

🎯 품질 명령어

/improve 증거 기반 코드 개선
/cleanup 기술 부채 정리
/test 테스팅 워크플로우

플래그 시스템

플래그는 명령어의 동작을 세밀하게 제어하고 최적화합니다.

🧠 사고 및 분석 플래그

--think 다중 파일 분석 (~4K 토큰)
--think-hard 깊은 아키텍처 분석 (~10K 토큰)
--ultrathink 중요 시스템 재설계 분석 (~32K 토큰)

⚡ 효율성 플래그

--uc 30-50% 토큰 압축
--validate 사전 검증 및 위험 평가
--safe-mode 최대 검증 보수적 실행

🌐 MCP 서버 플래그

--seq 복잡한 다단계 분석
--magic UI 컴포넌트 생성
--play 브라우저 자동화 및 테스팅

🔥 고급 활용

Wave 오케스트레이션

복잡한 작업을 다단계로 지능적으로 분해하여 실행하는 고급 기능입니다.

Wave 자동 활성화 조건

복잡도 ≥ 0.7
파일 수 > 20개
작업 유형 > 2개

Wave 전략

Progressive

점진적 개선을 위한 반복적 향상

Systematic

복잡한 문제를 위한 체계적 분석

Adaptive

다양한 복잡도 기반 동적 구성

Enterprise

100개 이상 파일의 대규모 오케스트레이션

Wave 사용 예제

# 대규모 시스템 개선 (자동 Wave 활성화)
/improve @entire-project/ --focus quality --scope system

# 결과:
# Wave 1: 현재 상태 분석
# Wave 2: 개선 계획 수립  
# Wave 3: 핵심 구조 개선
# Wave 4: 품질 검증 및 테스트
# Wave 5: 최종 최적화

MCP 서버 통합

4개의 전문 MCP 서버가 특화된 기능을 제공합니다.

📚 Context7

목적: 공식 라이브러리 문서화, 코드 예제, 모범 사례

자동 활성화: 외부 라이브러리 임포트, 프레임워크 질문

# Context7 활용 예제
/implement React Hook 폼 --c7
# → 공식 React 문서에서 패턴 검색
# → 모범 사례 적용
# → 버전 호환성 확인

🧠 Sequential

목적: 다단계 문제 해결, 아키텍처 분석, 체계적 디버깅

자동 활성화: 복잡한 디버깅, 시스템 설계, --think 플래그

# Sequential 활용 예제
/troubleshoot 성능 저하 이슈 --think-hard
# → 체계적 원인 분석
# → 가설 생성 및 검증
# → 단계별 해결 방안

✨ Magic

목적: 현대적 UI 컴포넌트 생성, 디자인 시스템 통합

자동 활성화: UI 컴포넌트 요청, 디자인 시스템 쿼리

# Magic 활용 예제
/implement 반응형 대시보드 --magic
# → 21st.dev 패턴 데이터베이스 검색
# → 접근성 컴플라이언스 보장
# → 모바일 우선 반응형 구현

🎭 Playwright

목적: 크로스 브라우저 E2E 테스팅, 성능 모니터링

자동 활성화: 테스팅 워크플로우, 성능 모니터링

# Playwright 활용 예제
/test e2e 사용자 로그인 플로우 --play
# → 다중 브라우저 테스트
# → 성능 메트릭 수집
# → 시각적 회귀 감지

멀티 에이전트 협업

복잡한 작업을 여러 전문 에이전트가 협력하여 병렬 처리합니다.

자동 위임 조건

  • 디렉토리 > 7개: --delegate --parallel-dirs
  • 파일 > 50개: --delegate files
  • 복잡도 > 0.8: --delegate auto
  • 도메인 > 3개: --delegate --parallel-focus

멀티 에이전트 예제

# 대규모 프로젝트 분석 (자동 위임)
/analyze @monorepo/ --comprehensive

# 자동 활성화된 에이전트들:
# Agent 1: Frontend 분석 (React 컴포넌트)
# Agent 2: Backend 분석 (API 엔드포인트)  
# Agent 3: 보안 분석 (인증/권한)
# Agent 4: 성능 분석 (병목점 식별)
# Agent 5: 품질 분석 (코드 품질)

# 성능 향상: 70% 시간 단축

트러블슈팅

🚨 일반적인 문제들

명령어가 인식되지 않음

원인: CLAUDE.md 파일이 올바른 위치에 없음

해결: ~/.claude/CLAUDE.md 파일 확인

페르소나가 자동 활성화되지 않음

원인: 키워드 매칭 또는 컨텍스트 분석 실패

해결: --persona-[name] 플래그로 수동 활성화

MCP 서버 연결 실패

원인: 서버 설정 또는 네트워크 문제

해결: --no-mcp로 네이티브 도구 사용

⚡ 성능 최적화

응답 시간이 느림

해결책:

  • --uc 플래그로 토큰 압축
  • --delegate로 병렬 처리
  • 불필요한 MCP 서버 비활성화
